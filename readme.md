# Телеграмм-бот для калымщиков и их клиентов

## Цель:
- Обеспечить взаимодействие заказчиков и работников
- Продумать платные услуги



## Функционал

### Калымщик
- Создаёт свой профиль
    - Информация
        - Фото
            - Выставлять только фото с собой
        - ФИО
        - Номер телефона
        - Предлагаемые услуги, прайс-лист
        - Территория обслуживания
        - Рейтинг (Обновляется один раз каждым, кроме подрядчика)
    - Технические моменты
        - Создать реплай-кнопку о предложении зарегистрировать свой профиль
        - Запрашивать через машину состояний
        - Хранить в базе данных
        - Через свой ID предоставить возможность создавать, редактировать, удалять свой профиль
- Предлагает услуги
    - Предлагать заказчикам по фильтрам услуг и территории обслуживания
- ведёт прерписку с клиентами
    - Организовать приватные комнаты

### Заказчик
- Размещает свой заказ
    - Информация
        - ФИО
        - Заголовок заказа
        - описание заказа
        - Примерная оплата заказа
        - Территория выполнения заказа
        - номер телефона
- Просматривает подрядчиков
    - Предложить фильтр по специализации и заказам
        - Создать реплай-кнопку для фильтрации подрядчиков
- Имеет возможность выбрать подрядчика
    - Создать инлаин-кнопки под каждым заказчиком
- Платная функция: заказчик видит рейтинг подрядчика
- Предложить после выполнения заказа поставить оценку подрядчику и оставить отзыв в свободном чате (включить фильтр ненормативной лексики)

### Особенности 
- Заказчик может быть подрядчиком, и подрядчик может быть заказчиком, поэтому:
    - Логика работы бота едина для всех, кроме разработчика
    - Первое время никакой модерации не будет, вполностью свободная пложадка для найма
    - Завести свободный чат, где каждый сможет оставить отзым о заказчике или подрядчике с возможностью поиска по тексту

## Примерный порядок работы и логика реализации:
**Подрядчик**
- Заполняет свою услугу
    - На время заполнения работает State
    - Со State данные передаются в базу данных
- После заполнения услуги
    - Будет показывать заказы по фильтрам услуги и территории
    - Под заказом будет кнопка "Взять заказ"
    - Надо предусмотреть согласие или отказ клиента на услугу
- После предложения и согласия 
    - Ограничить видимость услуги
    - Открывается приватный чат
    - Внизу чата реплай-кнопки "Отказаться от услуги" и "Услуга выполнена"
    - При отказе от услуги чат закрывается

## Платные услуги
### Аналог безопасной сделки
Цель:
- Сделать бот акупаемым
- Отсеять спам
- Отсеять пустые заявки и моженников
- Повысить доверие бота (можно договариваться и за пределами бота, но сервисный сбор будет говорить о серьёзных намерениях подрядчика)

Роли и ответственность
- Я предоставляю бота для составления заказов, услуг, связи между заказчиком и подрядчиком
- Я принимаю платежи через бота, тем самым отсеевая спам, мошенников, и формируя здоровые трудовые отнршения
    - За предоставление контактов подрядчика и заказчика
- Мотивирую это тем, что предоставляю приватный чат через бота с определёнными функциями
    - Подтвердить завершение работы
    - Отказаться от услуги
    - Оценить оказание услуги
